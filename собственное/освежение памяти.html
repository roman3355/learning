Картинка

Таблицы

Структура таблицы: заглавная строка таблицы вкладывается в контейнер thead, содержимое в контейнер tbody. Таблицы состоят не из столбцов, а из яйчеек <th> (заголовок) и <td> (содержимое). В свою очередь, эти яйчейки объединяются в строки с помощью контейнеров <tr>.

Формы

<Form> необходимы для создания форм для отправки данных. Внутри form распологается тег <input>, который является основополагающим для таких форм, так как он представляет собой контейнер, в который вводится информация для отправки на сервер. <input> может быть как примитивным полем без какой-либо специализации, так и узко специализированным (поля для ввода пароля, пола, радио-кнопки и т.д.). После создания <input>, ему можно присваивать имя и тип.

Отправка содержания форм

Содержание вписанных в теги <input> атрибутов name при отправке данных через get-запрос будут отображаться в ссылке, через которую данные будут отправлены на сервер. Содержимое <input> не будет отображаться в адресной строке, если в нём нет атрибута name.

placeholder

Атрибут placeholder="" вписывается в <input>, а его содержание показывается в поле вводы, исчезая сразу после начала ввода. Это необходимо для информирования пользователя.

label

<label> применяется для возможности сделать кликабельным название формы, то есть пользователь нажимает курсором на название и автоматически активируется поле для заполнения. <label> можно использовать двумя способами: либо поместить весь тег <input> в тег <label>; либо поместить в <label> только название тега <input>, а содержание оставить без изменения (в этом случае в открывающий тег <label> необходимо поместить атрибут for="", в содержании которого будет находиться название элемента id="", который должен быть помещён в открывающий тег <intput>, чтобы между ними образовалась связь - пример в коде с полем "Фамилия").

Проверка заполнения форм

Если необходимо, чтобы при отправке формы были обязательно заполнены, то в каждый <input> вписывается атрибут required.

select

Элемент <select> позволяет сделать выпадающий список. Этот список будет состоять из пунктов, каждый из которых должен быть вписан в тег <option>.


Видео, аудио, семантика

Видео

Видеофайлы размещаются в отдельном контейнере <video>. Свойства, которые должны быть у этого видеофайла, размещаются внутри открывающего тега <video>. Арибутами могут быть высота height, ширина width, элементы управления controls, предзагрузка видео preload (значение auto), автозапуск autoplay (должно быть значение true), повтор loop (должно быть значение loop), отключение звука muted (значение muted, это обязательное условие Google для загрузки видео).
В теге <source> размещается ссылка на видео src (="" в кавычках) и тип видео type (очень важный атрибут, записывается как type="video/mp4").
Ссылку на видео, размещённое на YouTube можно получить непосредственно на сайте, кликнув ПКМ на видео. Однако есть оговорка: src должен иметь в адресе ссылки слово embed, тогда видео будет воспроизводиться.


Аудио

С аудиофайлами алгоритм размещение почти такой же, как и с видео, но проще. Указывается тег <audio>, внутри которого пишется src для ссылки, тип аудио type (type="audio/mp3") и обязательный атрибут controls (без него файл будет недоступен). Также можно записать атрибуты автозапуск autoplay (должно быть значение true) и повтор loop (должно быть значение loop).

Семантические элементы

Семантические элементы HTML более специализированные, чем обычный элемент <div> и имеют свои собственные названия, которые описывают их специализацию.
Некоторые важные семантические элементы:

  • <article>
    • Элемент <article> (агл. "статья") определяет независимый, самодостаточный контент. Контент, помещенный в этот элемент, должен иметь смысл сам по себе, т. е. он должен быть понятен в отрыве от остальных частей веб-сайта. В качестве примеров использования элемента <article> могут выступать:
    • Публикация на форуме
    • Публикация в блоге
    • Газетная статья
  • <aside>
    • Элемент <aside> определяет некий контент, находящийся в стороне от контента, внутри которого он расположен (как боковой блок страницы, "сайдбар").
    Контент внутри элемента <aside> должен соотноситься с окружающим контентом.
  • <details>
    • Элемент <details> может использоваться для создания интерактивного виджета, который пользователь может открывать и закрывать. Внутри тега <details> можно размещать любой контент, тут полезные примеры
  • <figure>
    • Изображение и пояснение к нему сгруппировывается в элементе <figure>.
  • <figcaption>
    • Назначение элемента <figcaption> - добавление визуального пояснения к изображению. Пояснение записывается в виде <figcaption>Пояснение к изображению</figcaption> на следующей строке после строки <img>. Изображение и пояснение к нему должны быть сгруппированы внутри элемента <figure>.
  • <footer>
    • Элемент <footer> предназначен для определения "подвала" документа или раздела. Этот элемент должен содержать информацию о содержащем его элементе. Обычно в "подвале" размещают информацию об авторе документа, ссылки на условия использования текста, информация об авторских правах, контактные данные и т.п.
    В одном документе разрешается определять несколько элементов <footer>.
  • <header>
    • Элемент <header> предназначен для определения заголовочного блока или "шапки" документа или раздела. Элемент <header> следует использовать как контейнер для вводной информации.
    В одном документе разрешается определять несколько элементов <header>.
  • <main>
    • Тег <main> определяет основное содержимое документа, делает презентацию его назначения.
    Содержимое внутри элемента <main> должно быть уникальным для данного документа. Здесь не должно быть ничего, что повторяется где-либо, вроде сайдбаров, навигационных ссылок, информации об авторских правах, логотипов сайта, поисковых форм и т.п. В документе может быть только один элемент <main>, и он не должен быть вложен в элементы <article>, <aside>, <footer>, <header> или <nav>.
  • <mark>
  • <nav>
    • Элемент <nav> определяет набор ссылок навигации, однако следует помнить, что он предназначен только для основного блока навигационных ссылок.
  • <section>
    • Определяет раздел в документе. В соответствии со спецификацией W3C по HTML5: "Раздел — это тематически сгруппированный контент, как правило с заголовком." Домашняя страница обычно может быть разбита на следующие разделы: вступление, основной контент и контактная информация. То есть в <section> могут входить <h1> и <p>
  • <summary>
  • <time>
